# STRATEGY.md - 长期战略文件

## 长期目标

**员工数字孪生生产力系统**

构建一个能够映射、分析和优化员工工作模式的数字孪生系统，通过数据驱动的方式提升个人和团队生产力。

---

## 当前活跃项目及优先级

| 优先级 | 项目 | 状态 | 说明 |
|--------|------|------|------|
| P0 | 核心框架搭建 | 进行中 | 建立基础架构和数据管道 |
| P1 | 数据采集模块 | 待启动 | 整合多源工作数据 |
| P2 | 分析引擎 | 规划中 | 行为模式识别与洞察 |
| P3 | 可视化界面 | 规划中 | 用户友好的数据呈现 |

## 技术栈偏好

### 后端
- **语言**: Python (数据分析), Go (高性能服务)
- **数据库**: PostgreSQL (主数据), ClickHouse (时序数据)
- **消息队列**: Redis / NATS

### 前端
- **框架**: React + TypeScript
- **可视化**: D3.js / ECharts

### 基础设施
- **容器**: Docker + Kubernetes
- **CI/CD**: GitHub Actions
- **监控**: Prometheus + Grafana

## 工作模式偏好

### 验证 → 规范

1. **快速验证** (1-2周)
   - 构建最小可行原型
   - 验证核心假设
   - 收集早期反馈

2. **迭代优化** (2-4周)
   - 基于反馈调整方向
   - 完善关键功能
   - 技术债务控制

3. **规范化** (持续)
   - 文档完善
   - 测试覆盖
   - 性能优化
